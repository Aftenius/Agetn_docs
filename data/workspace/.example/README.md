# Пример workspace (без конфиденциальных данных)

Скопируйте содержимое в свой каталог (например `data/workspace/default` или путь из `DOCS_AGENT_WORKSPACE`) и замените плейсхолдеры на реальные требования и шаблоны.

- `settings.yaml` — название компании и инструкции для генерации структуры
- `requirements.md` — внутренние требования к договорам
- `checklist.json` — чек-лист (альтернатива DOCX «ФИНАЛЬНЫЙ ЧЕК-ЛИСТ.docx» в этой же папке)
- `templates/contracts/*.j2` — Jinja-шаблоны (необязательно, если хватает встроенных в репозитории)
- `agents/*.yaml` — переопределение агентов проверки по `id`
- `samples/` — примеры договоров для мастера (DOCX/PDF)
- `rag/inbox/` — сюда попадают загрузки для RAG; индекс в `rag_index/`

Каталог `default` под версионным контролем не хранится — только этот `.example`.
